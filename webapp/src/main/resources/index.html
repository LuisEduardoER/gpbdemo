gpbdemo

This project shows how to create RESTful web service using Jersey and Google Protocol Buffers as communication protocol. For easy testing and development, single endpoint (see net.jetensky.gpbdemo.service.MyResource#echo?) may accepts both JSON and Google protocol buffers inputs, producing also JSON or GPB outputs.

This enables to utilize easy to understand JSON format during development and testing and powerful, fast and size efficient GPB protocol for real work. All in single endpoint:

Links

  • /myresource - Sample Hello world! GET request returning text/plain
  • /myresource/student - Sample GET request (old way) returning JSON
  • /myresource/studentGPB - Sample GET request returning Google protocol buffer message
  • /myresource/studentGPBjson - GET request to endpoint capable both returning JSON and Google protocol buffer message (based on Accept header)
  • Test form - enables to call Google protocol buffer endpoint /myresource/echo by providing JSON input
